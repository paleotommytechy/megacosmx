# MegacosmX - High-Performance Talent Ecosystem

A React + Supabase prototype for connecting clients with verified talent.

## Setup Instructions

1. **Install Dependencies**
   ```bash
   npm install
   ```

2. **Environment Variables**
   Create a `.env` file in the root directory:
   ```env
   VITE_SUPABASE_URL=your_supabase_project_url
   VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
   ```
   *Note: If no env vars are provided, the app will gracefully degrade to use mock data for demonstration purposes.*

3. **Run Development Server**
   ```bash
   npm run dev
   ```

## Database Initialization (Supabase SQL)

Run the following SQL in your Supabase SQL Editor to set up the tables and security policies.

```sql
-- 1. Create Talents Table
create table public.talents (
  id uuid default gen_random_uuid() primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  name text not null,
  role text not null,
  skills text[] not null, -- Array of strings
  bio text not null,
  photo_url text,
  email text,
  whatsapp text
);

-- 2. Create Requests Table
create table public.requests (
  id bigint generated by default as identity primary key,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  talent_id uuid references public.talents(id) not null,
  client_name text not null,
  message text not null,
  status text default 'pending' check (status in ('pending', 'accepted', 'rejected'))
);

-- 3. Enable Row Level Security (RLS)
alter table public.talents enable row level security;
alter table public.requests enable row level security;

-- 4. RLS Policies

-- Policy: Everyone can view talents (Public Directory)
create policy "Enable read access for all users"
on public.talents for select
to public
using (true);

-- Policy: Everyone can submit a request (Public Submission)
create policy "Enable insert access for all users"
on public.requests for insert
to public
with check (true);

-- Policy: Only Authenticated Users (Admins) can view requests
-- Note: In a production app, you would refine this to specific admin roles
create policy "Enable read access for authenticated users only"
on public.requests for select
to authenticated
using (true);

-- 5. Insert Sample Data (Optional)
insert into public.talents (name, role, skills, bio, email, whatsapp)
values

```

## Features

- **Talent Gallery**: Grid layout with hover effects.
- **Filtering**: Filter talents by skills (using PostgreSQL array containment).
- **Direct Requests**: Modal form to submit hiring requests directly to Supabase.
- **Admin View**: Restricted dashboard to view incoming requests.
- **Mobile First**: Tap-to-call WhatsApp integration.
